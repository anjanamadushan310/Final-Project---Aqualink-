## üìã Quick Setup Guide

### Option 1: Use Placeholder Images
Create simple placeholder images with the correct names until you have real photos.

### Option 2: Rename Your Images
1. Create the directory structure as shown above
2. Rename your images to match the exact paths listed
3. Restart the backend to trigger seeding

### Option 3: Update Paths in Code
If you already have images with different names, modify the `DatabaseSeeder.java` file to match your existing image names.

## ‚úÖ Verification

After adding images, you can verify they're being loaded by:

1. **Starting the backend**
   ```bash
   cd Backend
   mvnw spring-boot:run
   ```

2. **Checking the database:**
   ```sql
   -- Fish ads with images
   SELECT id, name, image_paths FROM fish_images fi 
   JOIN fish_ads fa ON fi.fish_id = fa.id 
   LIMIT 5;

   -- Industrial equipment with images
   SELECT id, name, image_paths FROM industrial_images ii
   JOIN industrial_stuff ins ON ii.industrial_id = ins.id
   LIMIT 5;

   -- Services with images
   SELECT id, name, image_paths FROM service_images si
   JOIN services s ON si.service_id = s.id
   LIMIT 5;

   -- Blog posts with featured images
   SELECT id, title, featured_image_path FROM blog_posts LIMIT 5;

   -- User profile logos
   SELECT u.name, p.business_name, p.logo_path 
   FROM profile p 
   JOIN users u ON p.user_id = u.id
   WHERE p.logo_path IS NOT NULL;

   -- Banners
   SELECT banner_id, image_url FROM banner;
   ```

3. **Testing the frontend** to see if images display correctly
